    Chapitre 2.2 : L'Impact Mapping - Aligner le Produit sur les Objectifs Business

    🎯 Objectif de ce Chapitre link

    Ce chapitre vous donne une méthodologie complète pour utiliser l’Impact Mapping avant l’Event Storming. Vous apprendrez :

    • Comment aligner le produit sur les objectifs business
    • Les 4 niveaux de l’Impact Map (objectif, acteurs, impacts, livrables)
    • Comment éviter les écueils courants
    • L’intégration avec l’Event Storming et l’Example Mapping

    Qu’est-ce que l’Impact Mapping ? link

    Définition link

    L’Impact Mapping est une méthode de planification stratégique créée par Gojko Adzic qui aide à établir et visualiser les relations de causalité entre :

    • Un objectif business
    • Les acteurs impliqués
    • Les impacts attendus sur ces acteurs
    • Les livrables qui vont créer ces impacts

    Pourquoi l’Impact Mapping ? link

    1. Recentrer sur les Objectifs Business link

    Au lieu de partir d’une liste de fonctionnalités, l’Impact Mapping part des objectifs business et remonte vers les solutions.

    2. Favoriser la Collaboration link

    L’atelier implique business owners et équipes techniques pour créer une compréhension commune.

    3. Challenger les Hypothèses link

    La méthode encourage à définir et challenger les hypothèses sur lesquelles se base la stratégie.

    4. Intégration avec le Lean Startup link

    L’Impact Mapping s’inscrit parfaitement dans la Build-Measure-Learn Feedback Loop.

    Quand Utiliser l’Impact Mapping ? link

    ✅ Utilisez l’Impact Mapping si : link

    • Vous démarrez un nouveau projet
    • Vous voulez recentrer un projet existant
    • Les objectifs business ne sont pas clairs
    • Il y a un décalage entre business et technique
    • Vous voulez challenger une roadmap existante

    ❌ Évitez l’Impact Mapping si : link

    • Les objectifs business sont déjà très clairs
    • Le périmètre est très technique
    • L’équipe est déjà parfaitement alignée
    • Le temps est très limité

    Les 4 Niveaux de l’Impact Map link

    1. 🎯 L’Objectif (Le Pourquoi ?) link

    L’élément central de l’Impact Map est l’objectif stratégique poursuivi. La question clé est : “Pourquoi faisons-nous cela ?”

    Caractéristiques d’un Bon Objectif link

    • Mesurable : On peut quantifier le succès
    • Temps défini : Horizon de 3-6 mois
    • Business : Pas technique
    • Motivant : Inspire l’équipe

    Exemples de Bons Objectifs link

    ✅ Augmenter les revenus de 20% d'ici 6 mois
✅ Réduire le taux d'abandon de 30% d'ici 3 mois
✅ Améliorer la satisfaction client de 15% d'ici 4 mois

    Exemples de Mauvais Objectifs link

    ❌ Créer une application mobile
❌ Migrer vers React
❌ Améliorer les performances
❌ Être plus agile

    2. 👥 Les Acteurs (Le Qui ?) link

    Identifier qui peut avoir une influence sur le résultat. Les questions clés sont :

    • Sur le comportement de qui souhaitons-nous avoir un impact ?
    • Qui peut produire l’effet attendu ?
    • Qui peut nous aider à atteindre l’objectif ?
    • Qui peut nous empêcher ou nous freiner ?

    Types d’Acteurs link

    • Utilisateurs finaux : Ceux qui utilisent le produit
    • Influenceurs : Ceux qui influencent les utilisateurs
    • Décideurs : Ceux qui prennent les décisions d’achat
    • Opposants : Ceux qui peuvent freiner le projet

    Exemple : Système de Réservation Hôtelière link

    👥 ACTEURS :
- Voyageurs (utilisateurs finaux)
- Agences de voyage (influenceurs)
- Directeurs d'hôtel (décideurs)
- Concurrence (opposants)

    3. 🎯 Les Impacts (Le Comment ?) link

    Définir les changements de comportement souhaités chez les acteurs. La question clé est : “Comment pouvons-nous changer leur comportement ?”

    Caractéristiques d’un Bon Impact link

    • Comportemental : Décrit un changement d’action
    • Mesurable : On peut observer le changement
    • Spécifique : Pas vague ou général
    • Réaliste : Atteignable avec les moyens disponibles

    Exemples de Bons Impacts link

    ✅ Les voyageurs réservent plus souvent via notre plateforme
✅ Les agences recommandent notre hôtel en priorité
✅ Les directeurs d'hôtel renouvellent leur contrat
✅ La concurrence perd des parts de marché

    Exemples de Mauvais Impacts link

    ❌ Les utilisateurs sont satisfaits
❌ Le système est plus performant
❌ L'interface est plus intuitive
❌ Le code est plus maintenable

    4. 📦 Les Livrables (Le Quoi ?) link

    Définir les fonctionnalités et initiatives qui vont créer les impacts. La question clé est : “Que pouvons-nous faire pour produire ces impacts ?”

    Caractéristiques d’un Bon Livrable link

    • Actionnable : On peut l’implémenter
    • Lien clair : Connecté à un impact spécifique
    • Testable : On peut valider son efficacité
    • Priorisable : On peut le classer par importance

    Exemple : Système de Réservation Hôtelière link

    📦 LIVRABLES :
- Système de réservation en ligne
- Programme de fidélité
- Interface mobile optimisée
- Système de notation et avis
- Intégration avec les agences de voyage

    Guide Pratique de l’Atelier link

    Préparation de l’Atelier link

    1. Composer l’Équipe (5-6 personnes maximum) link

    • 1 Business Owner Senior : A le pouvoir de décision
    • 1-2 Représentants Techniques : Comprendent les contraintes
    • 1-2 Experts Métier : Connaissent les utilisateurs
    • 1 Facilitateur : Anime l’atelier

    2. Préparer le Matériel link

    • Tableau blanc ou mur
    • Post-its de 4 couleurs différentes
    • Marqueurs
    • Timer (pour respecter les phases)

    3. Définir la Durée link

    • 2-3 heures pour un atelier complet
    • 1 heure pour une session de validation

    Déroulement de l’Atelier link

    Phase 1 : Définir l’Objectif (20-30 min) link

    1. Le Business Owner présente le contexte
    2. L’équipe discute et clarifie l’objectif
    3. L’objectif est écrit au centre du tableau
    4. L’équipe valide que l’objectif est mesurable et motivant

    Phase 2 : Identifier les Acteurs (30-45 min) link

    1. L’équipe brainstorm sur tous les acteurs possibles
    2. Chaque acteur est écrit sur un post-it
    3. Les acteurs sont organisés autour de l’objectif
    4. L’équipe valide que tous les acteurs importants sont présents

    Phase 3 : Définir les Impacts (45-60 min) link

    1. Pour chaque acteur, l’équipe définit les impacts souhaités
    2. Chaque impact est écrit sur un post-it
    3. Les impacts sont placés sous les acteurs concernés
    4. L’équipe valide que les impacts sont comportementaux et mesurables

    Phase 4 : Identifier les Livrables (30-45 min) link

    1. Pour chaque impact, l’équipe identifie les livrables possibles
    2. Chaque livrable est écrit sur un post-it
    3. Les livrables sont placés sous les impacts concernés
    4. L’équipe valide que les livrables sont actionnables

    Exemple Complet : Plateforme de Formation link

    Objectif link

    🎯 OBJECTIF : Augmenter le nombre d'inscriptions aux formations de 50% d'ici 6 mois

    Acteurs Identifiés link

    👥 ACTEURS :
- Développeurs (utilisateurs finaux)
- Managers IT (décideurs d'achat)
- Influenceurs tech (recommandations)
- Concurrents (opposition)

    Impacts Souhaités link

    🎯 IMPACTS :
- Les développeurs s'inscrivent plus souvent aux formations
- Les managers IT budgétisent plus pour la formation
- Les influenceurs recommandent notre plateforme
- Les concurrents perdent des parts de marché

    Livrables Identifiés link

    📦 LIVRABLES :
- Système de recommandation personnalisé
- Programme de certification
- Contenu gratuit pour attirer
- Intégration avec les outils de développement
- Système de gamification
- Partenariats avec les influenceurs

    Écueils à Éviter link

    ❌ Trop de Participants link

    Problème : Plus de 6 personnes rendent l’atelier difficile à animer Solution : Limiter à 5-6 personnes maximum, inclure les bonnes personnes

    ❌ Ne Pas Impliquer les Bonnes Personnes link

    Problème : Inviter des personnes sans pouvoir de décision Solution : S’assurer que les business owners seniors sont présents

    ❌ Critiquer Trop Tôt link

    Problème : Tuer la créativité en critiquant les propositions Solution : Accorder du temps à la phase divergente avant de converger

    ❌ Post-Rationaliser une Liste de Fonctionnalités link

    Problème : Venir avec une liste pré-définie de fonctionnalités Solution : Utiliser cette liste comme point de départ, puis challenger

    ❌ Se Perdre dans les Détails link

    Problème : Vouloir tout détailler dès le premier atelier Solution : Se concentrer sur les acteurs et impacts, les livrables évolueront

    Intégration avec Event Storming et Example Mapping link

    Workflow Complet link

    1. Impact Mapping (2-3 heures) link

    • Définir les objectifs business
    • Identifier les acteurs et impacts
    • Prioriser les livrables

    2. Event Storming (2-4 heures) link

    • Découvrir les événements métier
    • Modéliser les processus
    • Définir les bounded contexts

    3. Example Mapping (30-45 min par fonctionnalité) link

    • Détailer les règles métier
    • Créer des exemples concrets
    • Identifier les cas limites

    Exemple de Workflow link

    Impact Mapping → "Augmenter les inscriptions de 50%"
     ↓
Event Storming → "FormationInscrite", "FormationComplétée"
     ↓
Example Mapping → Règles de validation des inscriptions
     ↓
Implémentation → Code métier robuste

    Avantages de l’Impact Mapping link

    1. Alignement Business-Technique link

    • Toute l’équipe comprend les objectifs
    • Les décisions techniques sont guidées par le business
    • Réduction des malentendus

    2. Priorisation Claire link

    • Les livrables sont connectés aux objectifs
    • Facilite la prise de décision
    • Évite le développement de fonctionnalités inutiles

    3. Communication Améliorée link

    • Langage commun entre business et technique
    • Vision partagée du projet
    • Facilite les présentations aux stakeholders

    4. Flexibilité link

    • Facilite les pivots si les hypothèses sont invalidées
    • Permet de réévaluer régulièrement la stratégie
    • S’intègre bien dans les méthodologies agiles
